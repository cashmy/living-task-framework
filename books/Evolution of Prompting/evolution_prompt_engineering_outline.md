## **Chapter 2 — Understanding Context Inference Prompting (CIP)**

### Giving Models Cognitive Context

- Definition: *Context Inference Prompting (CIP)* is the practice of embedding situational data—like markdown briefs, milestones, or narratives—so the AI infers purpose without explicit format instructions.
- Cognitive foundation: Humans think narratively; LLMs respond structurally.
- Example: Supplying a design brief rather than a command prompt.
- CIP as a bridge between *implicit intention* and *explicit structure*.

- *Tools & Practices:**

- Embedding markdown context.
- Context layers in VS Code, Docusaurus, and documentation workflows.
- Using “context blocks” as reusable cognitive scaffolds.

- The Stanford, Northeastern, and WVU study: *“Generate five responses with their probabilities.”*
- How Verbalized Sampling (VS) restores diversity without retraining.

- Sampling from the tails of the model’s probability distribution.

## **Chapter 4 — Cognitive Prompt Architecture (CPA)**

- *Architecture Principles:**

- Context → Distribution → Evaluation → Adaptation.
- Prompts as reusable cognitive modules.
- Integrating context graphs and probabilistic sampling in workflows.
