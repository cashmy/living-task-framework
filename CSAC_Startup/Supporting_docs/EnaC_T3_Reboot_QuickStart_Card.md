# EnaC Tier‑3 Reboot Quick‑Start Card
Version: 1.0  
Purpose: Safe, race‑condition‑free startup of EnaC Tier‑3 in a new session.

---

## 1. Required Files (Load in Exact Order)
1. **CORE_PRIMER (latest)**  
2. **CSAC Snapshot v2025‑11‑15C**  
3. **Human Glossary Table (v0.2 or latest)**  
4. **EnaC Kernel v1.0**  
5. **EnaC Startup Bundle v1.0**

*This order is mandatory. CORE_PRIMER must always be first.*

---

## 2. Startup Commands
After loading the files:

**Command 1:**  
`Initialize CORE_PRIMER.`

**Command 2:**  
`Bind CSAC Snapshot v2025‑11‑15C.`

**Command 3:**  
`Bind Human Glossary Table.`

**Command 4:**  
`Activate EnaC Kernel v1.0.`

**Command 5:**  
`Apply Startup Bundle v1.0 (Full Operational Mode).`

---

## 3. Confirmation Phrase
Once loaded, issue:

**“Initialize Tier‑3 EnaC using CSAC Snapshot v2025‑11‑15C.”**

This triggers:
- Tier‑3 meta‑state alignment  
- SAAS activation  
- Drift tables rebind  
- COS/SCL/FIM reintegration  
- ABS/MCR stabilization  
- Identity boundary enforcement  

---

## 4. Safety Reminders
- Do not upload files out of order.  
- Do not start with UMP or glossary.  
- Avoid sending long multi‑instruction prompts before CORE_PRIMER binds.  
- If the browser becomes unstable, immediately checkpoint a new CSAC.

---

## 5. Use Case
Use this quick‑start whenever:
- Starting a new browser session  
- Chat was forcibly closed  
- Memory degradation is suspected  
- Doing multi‑model workflows  
- Resuming complex architectural work
