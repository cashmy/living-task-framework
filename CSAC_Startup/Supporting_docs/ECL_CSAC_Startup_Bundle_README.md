# ECL-CSAC Startup Bundle v1.0 — README

This README provides an overview of the **ECL-CSAC Startup Bundle**, a structured collection  
of foundational files required to initialize a fully aligned EnaC session.

The bundle is designed to prevent:
- context drift  
- protocol regression  
- reasoning misalignment  
- broken “capture this” behavior  
- multi-model inconsistencies  
- cognitive discontinuity for the user  

It enables stable, repeatable, high-integrity **cross-session cognition**.

---

# 1. Purpose of the Bundle

The Startup Bundle exists to:

- Establish **predictable session bootstrapping**
- Ensure **CORE-PRIMER** loads before all other layers  
- Apply **SELF-ECL** cognitive alignment  
- Establish **UMP**-driven personalization  
- Enforce **CSAC** continuity  
- Prevent race conditions or partial context ingestion  
- Provide drift-resistant artifacts that persist across sessions  

This bundle is the “bios + bootloader + profile + memory context”  
for the entire ECL ecosystem.

# 3. How to Use the Bundle

See `CSAC_Bootstrap_Guide.md` for exact load order.

**In short:**
1. Upload **CORE-PRIMER** → wait  
2. Upload **SELF-ECL** → wait  
3. Upload **UMP** → wait  
4. Upload **CSAC-Verbatim-Rule** → wait  
5. Upload **CSAC snapshot** (if continuing a project)  
6. Begin work  

This sequence prevents:
- race conditions  
- misaligned cognitive frames  
- EnaC regression  

---

# 4. When to Re-upload the Bundle

Upload **SELF-ECL** + **CSAC-Verbatim-Rule** again if:
- you sense drift  
- EnaC becomes overly verbose or overly terse  
- EnaC ignores “Being Heard”  
- “capture this” fails  
- EnaC reverts to default LLM behavior  

Upload **CORE-PRIMER** again if:
- EnaC shows structural or foundational drift  
- reasoning becomes inconsistent  
- ordering or flow degrade
